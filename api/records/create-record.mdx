## POST /management

Creates a new record or updates an existing record in the management system. Records are uniquely identified by `NUMERO_DE_LA_ORDEN`.

<Note>
  This endpoint requires authentication with a management role. Include the JWT token in the Authorization header.
</Note>

### Authentication

```
Authorization: Bearer {token}
```

The authenticated user must have `profile.role === "management"` or the request will be denied.

### Request Body

<ParamField body="NUMERO_DE_LA_ORDEN" type="string" required>
  Unique order number that identifies the record
</ParamField>

<ParamField body="fecha_gestion" type="string" required>
  Management date. Accepts multiple formats: "DD/MM/YYYY", "YYYY/MM/DD", "D/M/YYYY", or ISO 8601
</ParamField>

<ParamField body="numero_producto" type="string">
  Product number
</ParamField>

<ParamField body="resultado_de_gestion" type="string">
  Management result/outcome
</ParamField>

<ParamField body="causal_de_pago" type="string">
  Payment reason/cause
</ParamField>

<ParamField body="causal_de_no_pago" type="string">
  Non-payment reason/cause
</ParamField>

<ParamField body="estado_servicio" type="string">
  Service status
</ParamField>

<ParamField body="recibo" type="string">
  Receipt number
</ParamField>

<ParamField body="valor_recibo" type="string">
  Receipt value/amount
</ParamField>

<ParamField body="lectura" type="string">
  Meter reading
</ParamField>

<ParamField body="tipo_de_vivienda" type="string">
  Housing type
</ParamField>

<ParamField body="contacto" type="string">
  Contact name
</ParamField>

<ParamField body="telefono_sugerido" type="string">
  Suggested phone number
</ParamField>

<ParamField body="observacion" type="string">
  Observations/notes
</ParamField>

<ParamField body="ubicacion" type="object">
  Location coordinates (lat/lng)
</ParamField>

<ParamField body="status" type="string">

</ParamField>

<ParamField body="uso_del_predio" type="string">
  Property usage
</ParamField>

<ParamField body="tipo_habitante" type="string">
  Resident type
</ParamField>

<ParamField body="clase_predio" type="string">
  Property class
</ParamField>

<ParamField body="tipo_comercio" type="string">
  Commerce type
</ParamField>

<ParamField body="nivel_estado_servicio" type="string">
  Service status level
</ParamField>

<ParamField body="cuota_inicial_ofrece" type="string">
  Initial quota offered
</ParamField>

<ParamField body="cuota_mensual" type="string">
  Monthly quota

  Number of people

</ResponseField>

## Behavior Notes

- **Upsert Logic**: If a record with the same `NUMERO_DE_LA_ORDEN` exists, it will be updated. Otherwise, a new record is created.
- **Date Handling**: The `fecha_gestion` is normalized to the start of day (midnight) in the server timezone
- **Auto-Update**: The `updatedAt` field is automatically set to the current date (start of day)
- **Field Updates**: All provided fields are set using `$set`, preserving any fields not included in the request
