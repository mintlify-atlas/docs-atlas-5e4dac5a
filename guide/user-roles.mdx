## Role Overview

Movilidad implements a hierarchical role-based access control (RBAC) system with three distinct roles:

<Steps>
  <Step title="Admin">
    **Highest Level** - Full system access
    
    Admins manage the entire system including user creation, manager assignments, and cross-project reporting.
  </Step>
  
  <Step title="Leader">
    **Middle Level** - Project management
    
    Leaders manage their specific project, upload records, assign field workers, and generate team reports.
  </Step>
  
  <Step title="Management (Gestor)">
    **Field Level** - Mobile operations
    
    Field managers use the mobile app to complete assigned service orders and upload evidence.
  </Step>
</Steps>

### Admin Methods

Methods restricted to admin role:

```javascript
Meteor.methods({
  getLeaders(userId) {
    const user = Meteor.users.findOne(userId);
    if (!user || user.profile.role !== 'admin') {
      throw new Meteor.Error('Unauthorized');
    }
    return Meteor.users.find({ 'profile.role': 'leader' }).fetch();
  },
  
  getAllManagers(userId) {
    const user = Meteor.users.findOne(userId);
    if (!user || user.profile.role !== 'admin') {
      throw new Meteor.Error('Unauthorized');
    }
    return Meteor.users.find({ 'profile.role': 'management' }).fetch();
  },
  
  assignManagersToLeader(leaderId, managerIds) {
    // Only admins can assign managers
    if (!this.userId) throw new Meteor.Error('Not logged in');
    
    const admin = Meteor.users.findOne(this.userId);
    if (admin.profile.role !== 'admin') {
      throw new Meteor.Error('Unauthorized');
    }
    
    Meteor.users.update(leaderId, {
      $set: { 'profile.managers': managerIds }
    });
  }
});
```

### Leader Methods

Methods available to leaders:

```javascript
Meteor.methods({
  async getRecords(page, pageSize, filters, sort) {
    // Automatically filter by user's project
    const project = Meteor.users.findOne(this.userId).profile.project;
    let filter = { project, ...filters };
    
    // Leaders can only see their own records
    if (Meteor.user().profile.role === 'leader') {
      filter.leaderId = this.userId;
    }
    
    // ... fetch and return records
  },
  
  async createRecord(record, leaderId) {
    // Verify leader is creating their own records
    if (this.userId !== leaderId) {
      throw new Meteor.Error('Unauthorized');
    }
    
    const _id = Random.id(10);
    return recordsCollection.insert({ ...record, leaderId, _id });
  },
  
  async updateRecordManager(recordId, managerId, leaderId, tipo_producto) {
    // Verify record belongs to this leader
    const record = recordsCollection.findOne({ 
      NUMERO_DE_LA_ORDEN: recordId, 
      leaderId 
    });
    
    if (!record) {
      throw new Meteor.Error('Record not found or access denied');
    }
    
    // Update manager assignment
    return await recordsCollection.updateAsync(
      { NUMERO_DE_LA_ORDEN: recordId, leaderId },
      { $set: { GESTOR: managerId, status: 'pending' } }
    );
  },
  
  getManagers(leaderId) {
    // Leaders can only get their own managers
    if (this.userId !== leaderId) {
      throw new Meteor.Error('Unauthorized');
    }
    
    const user = Meteor.users.findOne(leaderId);
    const managers = user.profile?.managers || [];
    
    return managers.map(managerId => {
      const manager = Meteor.users.findOne(managerId);
      return { username: manager.username, id: manager._id };
    });
  }
});
```

## Role Hierarchy

```
Admin
  │
  ├── Can create Leaders and Managers
  ├── Can assign Managers to Leaders
  ├── Can view all data
  └── System-wide access

Leader
  │
  ├── Manages specific project
  ├── Uploads service records
  ├── Assigns work to Managers
  └── Views team reports

Management (Gestor)
  │
  ├── Receives assignments from Leader
  ├── Completes field work
  ├── Submits results via mobile
  └── Limited web access
```

### Data Filtering by Project

All queries automatically filter by project:

```javascript
const project = Meteor.users.findOne(this.userId).profile.project;
const filter = { project, ...additionalFilters };

const records = recordsCollection.find(filter).fetch();
```

<Warning>
Users can NEVER see data from projects they're not assigned to. This is enforced at the method level.
</Warning>

## Role Detection

Detect user role in components:

```javascript
import useRole from './hooks/useRole';

function MyComponent() {
  const { userRole, isLoading } = useRole();
  
  if (isLoading) return <Spinner />;
  
  return (
    <div>
      {userRole === 'admin' && <AdminTools />}
      {userRole === 'leader' && <LeaderTools />}
      {userRole === 'management' && <ManagerInfo />}
    </div>
  );
}
```

### Example Security Pattern

```javascript
Meteor.methods({
  sensitiveOperation(data) {
    // 1. Check if user is logged in
    if (!this.userId) {
      throw new Meteor.Error('not-authorized', 'Must be logged in');
    }
    
    // 2. Get user and verify role
    const user = Meteor.users.findOne(this.userId);
    if (!user || user.profile.role !== 'admin') {
      throw new Meteor.Error('not-authorized', 'Admin access required');
    }
    
    // 3. Validate input
    check(data, {
      field1: String,
      field2: Number
    });
    
    // 4. Perform operation
    return performSensitiveOperation(data);
  }
});
```
