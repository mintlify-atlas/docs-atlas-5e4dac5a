## Overview

The Photo model uses the `meteor/ostrio:files` FilesCollection package to store photo files attached to management records. Photos are stored on the filesystem and metadata is stored in MongoDB.

## Configuration

<ResponseField name="collectionName" type="string">
  MongoDB collection name: `"photos"`
</ResponseField>

<ResponseField name="storagePath" type="string">
  Filesystem path where photos are stored: `"/data/photos"`
</ResponseField>

## File Validation

### Size Limit

Maximum file size: **10MB** (10,485,760 bytes)

### Accepted Formats

Only image files are accepted:
- PNG (`.png`)
- JPEG (`.jpg`, `.jpeg`)

Files that don't match these formats will be rejected with the error:
```
Formato no válido, solo se aceptan imagenes
```

## Photo Metadata Schema

When a photo is uploaded, the following metadata is stored:

<ResponseField name="fileId" type="string">
  Unique 10-character identifier generated using `Random.secret(10)`
</ResponseField>

<ResponseField name="fileName" type="string">
  Original filename from the upload (e.g., `"photo1.jpg"`)
</ResponseField>

<ResponseField name="type" type="string">
  MIME type of the file (e.g., `"image/jpeg"`, `"image/png"`)
</ResponseField>

<ResponseField name="size" type="number">
  File size in bytes
</ResponseField>

<ResponseField name="path" type="string">
  Full filesystem path to the stored file
</ResponseField>

<ResponseField name="_id" type="string">
  MongoDB document ID (auto-generated by FilesCollection)
</ResponseField>

## Storage in Records

Photos are linked to records through the `fotos` array field in the [Record Model](/api/models/record):

```json
{
  "NUMERO_DE_LA_ORDEN": "ORD-12345",
  "fotos": ["abc1234567", "xyz9876543"],
  ...
}
```

Each element in the `fotos` array is a `fileId` that references a photo document.

## Upload Process

1. Client sends multipart/form-data request to `/management/photos/:orden`
2. Server validates file size and format
3. File is stored to `/data/photos` directory
4. Metadata is saved to `photos` collection
5. Record is updated with the fileId(s) in the `fotos` array

## Example Photo Document

```json
{
  "_id": "mongodb_generated_id",
  "fileId": "abc1234567",
  "fileName": "exterior_photo.jpg",
  "type": "image/jpeg",
  "size": 2457600,
  "path": "/data/photos/abc1234567-exterior_photo.jpg",
  "uploadedAt": "2024-03-15T10:30:00.000Z",
  "versions": {
    "original": {
      "path": "/data/photos/abc1234567-exterior_photo.jpg",
      "size": 2457600,
      "type": "image/jpeg",
      "extension": "jpg"
    }
  }
}
```

## Error Handling

### File Too Large

When a file exceeds 10MB:
```
El tamaño maximo del archivo es de 10Mb
```

### Invalid Format

When a file is not PNG/JPG/JPEG:
```
Formato no válido, solo se aceptan imagenes
```

### Upload Failure

When the upload process fails:
```
Error uploading files
```

## Usage with API

See the [Upload Photos](/api/records/upload-photos) endpoint for details on uploading photos via the REST API.

## Implementation Details

- Uses Multer for handling multipart/form-data
- Files are stored in memory temporarily before being written to disk
- Up to 2 photos can be uploaded per request (foto1 and foto2)
- Storage uses `multer.memoryStorage()` strategy
