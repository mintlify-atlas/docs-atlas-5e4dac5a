## Introduction

The Movilidad mobile app is an Android application designed for field managers (gestores) to complete utility service visits, record results, and upload evidence from the field.

<Info>
The mobile app is currently available for Android only. The APK file can be downloaded directly from the web dashboard.
</Info>

## Mobile Architecture

The mobile app communicates with the Movilidad server via REST API:

```
[Mobile App] ←→ HTTPS/REST API ←→ [Meteor Server]
     │                                      │
     │                                      │
  [Local DB]                            [MongoDB]
```

### Communication Flow

<Steps>
  <Step title="Authentication">
    Manager logs in with username and password
    
    ```bash
    POST /login
    { "username": "manager1", "password": "pass" }
    
    Response: { "token": "eyJhbG..." }
    ```
  </Step>
  
  <Step title="Fetch Assignments">
    App retrieves assigned service orders
    
    ```bash
    GET /management/manager1
    Authorization: Bearer eyJhbG...
    
    Response: Array of service orders
    ```
  </Step>
  
  <Step title="Field Work">
    Manager visits location, completes inspection, fills out form
  </Step>
  
  <Step title="Submit Data">
    Submit visit results to server
    
    ```bash
    POST /management
    Authorization: Bearer eyJhbG...
    Body: { visit data }
    ```
  </Step>
  
  <Step title="Upload Photos">
    Upload photos taken during visit
    
    ```bash
    POST /management/photos/ORDER-123
    Authorization: Bearer eyJhbG...
    Files: foto1, foto2
    ```
  </Step>
</Steps>

## Data Fields

When submitting visit data, the mobile app collects:

<Tabs>
  <Tab title="Required Fields">
    - `NUMERO_DE_LA_ORDEN`: Order number (identifies the record)
    - `fecha_gestion`: Visit date (DD/MM/YYYY format)
    - `status`: Must be set to "reviewed" when completing
  </Tab>
  
  <Tab title="Visit Results">
    - `resultado_de_gestion`: Visit outcome
    - `causal_de_pago`: Payment reason (if applicable)
    - `causal_de_no_pago`: Non-payment reason
    - `estado_servicio`: Service status
  </Tab>
  
  <Tab title="Location Data">
    - `ubicacion.latitud`: GPS latitude
    - `ubicacion.longitud`: GPS longitude
    
    Automatically captured from device GPS.
  </Tab>
  
  <Tab title="Property Info">
    - `tipo_de_vivienda`: Housing type
    - `uso_del_predio`: Property use
    - `tipo_habitante`: Resident type
    - `clase_predio`: Property class
    - `tipo_comercio`: Commerce type (if commercial)
  </Tab>
  
  <Tab title="Additional Data">
    - `contacto`: Contact person
    - `telefono_sugerido`: Suggested phone number
    - `observacion`: Observations/notes
    - `recibo`: Receipt number
    - `valor_recibo`: Receipt value
    - `lectura`: Meter reading
  </Tab>
</Tabs>

## Security

### JWT Authentication

The mobile app uses JSON Web Tokens for authentication:

```javascript
const JWT_SECRET = 'your_jwt_secret_key';

// Token generation on login
const token = jwt.sign({ userId: user._id }, JWT_SECRET);

// Token verification on each request
jwt.verify(token, JWT_SECRET, (err, decoded) => {
  if (err) {
    // Token invalid or expired
  } else {
    // Token valid, userId = decoded.userId
  }
});
```

### Authentication Middleware

All protected endpoints verify:

1. **Token presence**: Authorization header must exist
2. **Token validity**: Token must be valid and not expired
3. **User existence**: User ID from token must exist in database
4. **Role verification**: User must have 'management' role

```javascript
function authenticate(req, res, next) {
  const authHeader = req.headers['authorization'];
  const token = authHeader && authHeader.split(' ')[1];

  if (!token) {
    res.writeHead(401);
    res.end('Unauthorized: No token provided');
    return;
  }

  jwt.verify(token, JWT_SECRET, (err, decoded) => {
    if (err) {
      res.writeHead(403);
      res.end(JSON.stringify({ message: 'Invalid token' }));
      return;
    }

    const user = Meteor.users.findOne(decoded.userId);
    if (!user || user.profile.role !== 'management') {
      res.writeHead(403);
      res.end(JSON.stringify({ message: 'Access denied' }));
      return;
    }

    next();
  });
}
```

<Warning>
**Security Note**: The JWT_SECRET must be changed from the default value before deploying to production.
</Warning>

<Steps>
  <Step title="Enable Unknown Sources">

  </Step>

  </Step>

    Open the Movilidad app from your app drawer
  </Step>
</Steps>

<Info>
The app is not available on Google Play Store and must be installed manually via APK.
</Info>

## Offline Capability

The mobile app is designed to work offline:

1. **Data Sync**: Assigned records are downloaded and stored locally
2. **Form Submission**: Visit data is queued when offline
3. **Photo Storage**: Photos are stored locally until upload
4. **Auto Sync**: When connection is restored, data syncs automatically

<Note>
Managers should sync data regularly when they have internet connectivity to avoid data loss.
</Note>
