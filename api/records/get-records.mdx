## GET /management/:username

Retrieves all management records assigned to a specific user (identified by username).

<Note>
  This endpoint requires authentication with a management role. Include the JWT token in the Authorization header.
</Note>

### Authentication

```
Authorization: Bearer {token}
```

The authenticated user must have `profile.role === "management"` or the request will be denied.

### Path Parameters

<ParamField path="username" type="string" required>
  The username of the manager (GESTOR) whose records you want to retrieve
</ParamField>

### Response

<ResponseField name="records" type="array">
  Array of record objects assigned to the specified user. See the [Record Model](/api/models/record) for field details.
</ResponseField>

### Error Responses

<ResponseField name="401 Unauthorized">
  Returned when no token is provided or system is in maintenance mode
  ```
  Unauthorized: No token provided
  ```
</ResponseField>

<ResponseField name="403 Forbidden">
  Returned when token is invalid or user doesn't have management role
  ```json
  {
    "message": "Invalid token"
  }
  ```
  or
  ```json
  {
    "message": "Access denied"
  }
  ```
</ResponseField>

### Example Request

```bash
curl -X GET https://your-domain.com/management/jperez \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

## Query Behavior

- Returns all records where `GESTOR` field matches the provided username
- Returns an empty array `[]` if no records are found for the user
- Records include all fields stored in the database (see [Record Model](/api/models/record))

## Use Cases

- **Mobile App**: Fetch all records assigned to the logged-in manager for offline access
- **Dashboard**: Display pending and completed records for a specific user
- **Reporting**: Retrieve user-specific records for performance analysis
