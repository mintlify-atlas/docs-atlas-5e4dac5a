## System Overview

Movilidad is built on a modern full-stack JavaScript architecture using Meteor.js, providing real-time data synchronization between web and mobile clients.

## Technology Stack

### Backend Framework

**Meteor 3.0.1**

Movilidad uses Meteor as its full-stack framework, providing:

- Isomorphic JavaScript (same code runs on client and server)
- Built-in WebSocket support for real-time updates
- Integrated MongoDB driver
- Hot Module Replacement for development
- Production-ready build system

```javascript
// server/main.js
import { Meteor } from 'meteor/meteor';
import settings from '../settings.json';

import '../imports/api/records/recordsIndex';
import '../imports/api/photos/photosIndex';
import '../imports/api/roles/rolesMethods';
import '../imports/api/documents/documentsIndex';

Meteor.startup(async () => {
  // Initialize admin user if not exists
  const admin = Meteor.users.findOne({ 
    username: settings.private.admin.user 
  });
  if (!admin) {
    Accounts.createUser({
      username: settings.private.admin.user,
      password: settings.private.admin.password,
      profile: { role: 'admin' }
    });
  }
});
```

## API Layer

### Meteor Methods

Server-side methods provide the application's business logic:

```javascript
// imports/api/records/recordsMethods.js
import { Meteor } from 'meteor/meteor';
import { recordsCollection } from './recordsCollection';

Meteor.methods({
  async getRecords(page, pageSize, filters, sort) {
    const project = Meteor.users.findOne(this.userId).profile.project;
    
    let filter = { project };
    if (filters) {
      for (const [key, value] of Object.entries(filters)) {
        filter[key] = key === 'GESTOR' 
          ? { $eq: value }
          : { $regex: value, $options: 'i' };
      }
    }
    
    const records = await recordsCollection
      .rawCollection()
      .aggregate([
        { $match: filter },
        { $sort: { [sort.sortField]: sort.sortOrder } },
        {
          $facet: {
            metadata: [{ $count: 'totalCount' }],
            data: [
              { $skip: (page - 1) * pageSize },
              { $limit: pageSize }
            ]
          }
        }
      ])
      .toArray();
    
    return {
      data: records[0].data,
      total: records[0].totalCount
    };
  }
});
```

<Info>
All methods automatically filter data by the user's project, ensuring data isolation.
</Info>

### REST API Endpoints

For mobile app integration, REST endpoints are provided via `meteorhacks:picker`:

```javascript
// imports/api/records/recordsEndpoints.js
import { Picker } from 'meteor/meteorhacks:picker';
import jwt from 'jsonwebtoken';

const JWT_SECRET = 'your_jwt_secret_key';

// POST /login - Authenticate and get token
postRoutes.route('/login', function(params, req, res) {
  const { username, password } = req.body;
  const user = Meteor.users.findOne({ username });
  
  if (!user) {
    res.writeHead(401);
    res.end(JSON.stringify({ message: 'Invalid credentials' }));
    return;
  }
  
  const result = Accounts._checkPassword(user, password);
  if (result.error) {
    res.writeHead(401);
    res.end(JSON.stringify({ message: 'Invalid credentials' }));
    return;
  }
  
  const token = jwt.sign({ userId: user._id }, JWT_SECRET);
  res.writeHead(200);
  res.end(JSON.stringify({ token }));
});
```

**Available Endpoints:**

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/login` | Authenticate user and get JWT token |
| POST | `/refresh-token` | Refresh expired JWT token |
| GET | `/management/:username` | Get assigned records for manager |
| POST | `/management` | Submit field visit results |
| POST | `/management/photos/:orden` | Upload photos for service order |
| GET | `/health` | Health check endpoint |

## Data Flow

### Web Application Flow

<Steps>
  <Step title="User Login">
    User enters credentials → `Meteor.loginWithPassword()` → Session created
  </Step>
  <Step title="Role Detection">
    `useRole()` hook reads `Meteor.user().profile.role` → Menu renders based on role
  </Step>
  <Step title="Data Subscription">
    Components call `Meteor.call()` methods → Server filters by project → Data returned
  </Step>
  <Step title="UI Update">
    `useTracker()` hook makes components reactive → UI updates automatically on data changes
  </Step>
</Steps>

### Mobile Application Flow

<Steps>
  <Step title="Authentication">
    POST `/login` → Receive JWT token → Store in device
  </Step>
  <Step title="Fetch Records">
    GET `/management/:username` with Bearer token → Receive assigned records
  </Step>
  <Step title="Field Visit">
    Manager visits location → Fills out form → Captures photos
  </Step>
  <Step title="Submit Data">
    POST `/management` with visit data → POST `/management/photos/:orden` with images
  </Step>
  <Step title="Status Update">
    Record status changes to 'reviewed' → Appears in reports
  </Step>
</Steps>

## Security Architecture

### Authentication

<Tabs>
  <Tab title="Web (Meteor Accounts)">
    - Built-in Meteor accounts system
    - Password hashing with bcrypt
    - Session management via DDP
    - Automatic logout on token expiration
  </Tab>
  <Tab title="Mobile (JWT)">
    - JSON Web Tokens for stateless auth
    - Token refresh mechanism
    - 24-hour token expiration
    - Bearer token in Authorization header
  </Tab>
</Tabs>

### Authorization

```javascript
// Role-based method authorization
Meteor.methods({
  getLeaders(userId) {
    const user = Meteor.users.findOne(userId);
    if (!user || user.profile.role !== 'admin') {
      throw new Meteor.Error('Unauthorized');
    }
    return Meteor.users.find({ 'profile.role': 'leader' }).fetch();
  }
});
```

### Data Isolation

All queries automatically filter by project:

```javascript

const filter = { project, ...additionalFilters };
```

### Docker Containerization

### Frontend Optimization

- **Code Splitting**: Dynamic imports for large components
- **Memoization**: React hooks prevent unnecessary re-renders
- **Virtual Scrolling**: Large tables rendered efficiently
- **Debounced Search**: Search inputs debounced to reduce queries

### File Storage

Photos stored on filesystem at `/data/photos` with database references:

```javascript
export const photosCollection = new FilesCollection({
  collectionName: 'photos',
  storagePath: '/data/photos'
});
```

<Warning>
Ensure the `/data/photos` directory has appropriate permissions and is backed up regularly.
</Warning>

## Scalability

Movilidad is designed to scale:

- **Horizontal Scaling**: Multiple Meteor instances behind load balancer
- **Database Sharding**: MongoDB supports sharding by project
- **File Storage**: Can be migrated to S3 or similar cloud storage
- **Caching**: Redis can be added for session storage and caching
