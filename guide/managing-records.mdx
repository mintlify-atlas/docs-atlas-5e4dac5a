## Overview

Records are the core data entities in Movilidad, representing utility service orders that need field management. Leaders upload records via Excel, assign them to field managers (gestores), and track their progress.

## Record Structure

Each record contains comprehensive utility service information:

<Tabs>
  <Tab title="Customer Information">
    - `NOMBRE_CLIENTE`: Customer name
    - `IDENTIFICACION`: Customer ID number
    - `DIRECCION_PREDIO`: Service address
    - `DESCRIPCION_BARRIO`: Neighborhood
    - `DESCRIPCION_CICLO`: Billing cycle
  </Tab>
  
  <Tab title="Service Details">
    - `NUMERO_DE_LA_ORDEN`: Order number (unique)
    - `CONTRATO`: Contract number
    - `PRODUCTO`: Product ID
    - `DESCRIPCION_TIPO_PRODUCTO`: Product type (Gas, Brilla, etc.)
    - `DESCRIPCION_CATEGORIA`: Service category
    - `DESCRIPCION_SUBCATEGORIA`: Subcategory/stratum
    - `ELEMENTO_MEDICION`: Meter number
    - `ULTIMA_LECTURA_TOMADA`: Last meter reading
  </Tab>
  
  <Tab title="Financial Data">
    - `DEUDA_TOTAL_ASIGNADA`: Total assigned debt
    - `TOTAL_DEUDA_CORRIENTE`: Current debt total
    - `CORRIENTE_VENCIDA_ASIGNADA`: Overdue current debt
    - `DIAS_DEUDA_ASIGNACION`: Days in debt at assignment
    - `EDAD_MORA_ACTUAL`: Current delinquency age
    - `ESTADO_FINANCIERO`: Financial state
    - `INDICADOR`: Status indicator (Normalización, Contención, Castigado)
  </Tab>
  
  <Tab title="Assignment Data">
    - `GESTOR`: Assigned manager ID
    - `leaderId`: Leader who owns the record
    - `project`: Project name (movilidad/sierra)
    - `period`: Assignment period
    - `fecha_asignacion`: Assignment date
    - `status`: Record status (pending/reviewed)
  </Tab>
  
  <Tab title="Field Visit Data">
    - `fecha_gestion`: Visit date
    - `resultado_de_gestion`: Visit result
    - `causal_de_pago`: Payment reason
    - `causal_de_no_pago`: Non-payment reason
    - `estado_servicio`: Service status
    - `observacion`: Observations
    - `ubicacion`: {latitud, longitud}
    - `fotos`: Array of photo IDs
    - `updatedAt`: Last update timestamp
  </Tab>
</Tabs>

### Create Record Method

Records are created using the `createRecord` method:

```javascript imports/api/records/recordsMethods.js
import { Random } from 'meteor/random';
import { recordsCollection } from './recordsCollection';

Meteor.methods({
  async createRecord(record, leaderId) {
    const _id = Random.id(10);
    
    // Remove duplicates by order number
    recordsCollection.remove({
      NUMERO_DE_LA_ORDEN: record.NUMERO_DE_LA_ORDEN
    });

    return recordsCollection.insert({ 
      ...record, 
      leaderId, 
      _id 
    });
  }
});
```

<Warning>
Duplicate order numbers are automatically removed and replaced with the new data.
</Warning>

## Viewing and Searching Records

### Records Table

Leaders view their records in a comprehensive table:

```javascript imports/ui/components/records/index.jsx
const fetchRecords = (page, pageSize, search, sort) => {
  Meteor.call(
    'getRecords',
    page,
    pageSize,
    { ...search, leaderId: Meteor.userId() },
    sort,
    (err, resp) => {
      if (!err) {
        setDataSource(resp.data);
        setPagination(prev => ({
          ...prev,
          current: page,
          total: resp.total
        }));
      }
    }
  );
};
```

### Update Record Manager Method

```javascript imports/api/records/recordsMethods.js
import moment from 'moment';

Meteor.methods({
  async updateRecordManager(recordId, managerId, leaderId, tipo_producto) {
    const product = {};
    if (tipo_producto) {
      product.DESCRIPCION_TIPO_PRODUCTO = tipo_producto;
    }

    return await recordsCollection.updateAsync(
      { NUMERO_DE_LA_ORDEN: recordId, leaderId },
      {
        $set: {
          ...product,
          GESTOR: managerId,
          status: 'pending',
          updatedAt: '',
          fecha_gestion: '',
          fecha_asignacion: moment().format('DD/MM/YYYY')
        }
      },
      { upsert: true }
    );
  }
});
```

<Info>
When a record is assigned:
- `GESTOR` field is set to manager's user ID
- `status` is set to 'pending'
- `fecha_asignacion` is set to current date
- Previous visit data is cleared
</Info>

## Pagination

Records are paginated for performance:

```javascript
const handleTableChange = (pagination) => {
  fetchRecords(pagination.current, pagination.pageSize, searchKeys);
  setPagination(pagination);
};
```

The `getRecords` method uses MongoDB aggregation for efficient pagination:

```javascript
Meteor.methods({
  async getRecords(page, pageSize, filters, sort) {
    page = parseInt(page, 10) || 1;
    pageSize = parseInt(pageSize, 10) || 50;
    const project = Meteor.users.findOne(this.userId).profile.project;

    let filter = { project };
    // Add search filters...

    const records = await recordsCollection
      .rawCollection()
      .aggregate([
        { $match: filter },
        { $sort: { [sortField]: sortOrder } },
        {
          $facet: {
            metadata: [{ $count: 'totalCount' }],
            data: [
              { $skip: (page - 1) * pageSize },
              { $limit: pageSize }
            ]
          }
        },
        {
          $project: {
            data: 1,
            totalCount: { $arrayElemAt: ['$metadata.totalCount', 0] }
          }
        }
      ])
      .toArray();

    return {
      data: records[0].data,
      total: records[0].totalCount
    };
  }
});
```

<Note>
The `$facet` stage allows getting both paginated data and total count in a single query.
</Note>

## Number Formatting

Financial fields are automatically formatted:

```javascript imports/ui/utils/formatIfNumber.js
export default function formatIfNumber(value) {
  // Check if value is a number
  const num = Number(value);
  
  if (!isNaN(num) && value !== '' && value !== null) {
    return new Intl.NumberFormat('es-CO', {
      minimumFractionDigits: 0,
      maximumFractionDigits: 2
    }).format(num);
  }
  
  return value;
}
```

Usage in table columns:

```javascript
render: (text) => formatIfNumber(text)
```

Excludes non-financial fields:

```javascript
const excludedFields = ['NUMERO_DE_LA_ORDEN', 'CONTRATO', 'PRODUCTO'];

render: (text, record) => 
  excludedFields.includes(dataIndex) 
    ? text 
    : formatIfNumber(text)
```

## Troubleshooting

<Accordion title="Excel import fails">
  Check:
  1. File is valid `.xlsx` or `.xls` format
  2. Column names exactly match expected fields
  3. NUMERO_DE_LA_ORDEN column exists and contains unique values
  4. File size is reasonable (< 50MB)
</Accordion>

<Accordion title="Cannot assign manager to record">
  Verify:
  1. You are logged in as a leader
  2. The manager is assigned to you (check via Gestores menu)
  3. Both you and the manager are in the same project
  4. The record belongs to your project
</Accordion>

<Accordion title="Search returns no results">
  Try:
  1. Clear all filters and search again
  2. Check for typos in search term
  3. Verify the field contains the expected data
  4. Reset pagination to page 1
</Accordion>
