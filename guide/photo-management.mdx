## Overview

Movilidad allows field managers (gestores) to upload photos as evidence of their field visits. Photos are uploaded from the mobile app and associated with service orders.

<Info>
Each service order can have up to 2 photos. Photos are stored on the server filesystem and referenced in the database.
</Info>

## Photo Collection

Photos are managed using the `ostrio:files` package:

```javascript imports/api/photos/photosCollection.js
import { FilesCollection } from 'meteor/ostrio:files';

function checkSize(size) {
  return size <= 10485760; // 10MB max
}

function checkFormat(regex_formats, file) {
  return regex_formats.test(file.extension);
}

function checkFile(size, regex_formats, file) {
  if (!checkSize(size)) {
    return 'El tamaño máximo del archivo es de 10Mb';
  }
  if (!checkFormat(regex_formats, file)) {
    return 'Formato no válido, solo se aceptan imagenes';
  }
  return true;
}

export const photosCollection = new FilesCollection({
  collectionName: 'photos',
  storagePath: '/data/photos',
  onBeforeUpload(file) {
    return checkFile(10485760, /png|jpg|jpeg/i, file);
  }
});
```

## Uploading Photos (Mobile App)

Field workers upload photos via the mobile API after completing a field visit.

### Upload Endpoint

```javascript imports/api/records/recordsEndpoints.js
import { Picker } from 'meteor/meteorhacks:picker';
import { Random } from 'meteor/random';
import multer from 'multer';
import { photosCollection } from '../photos/photosCollection';
import { recordsCollection } from './recordsCollection';

const storage = multer.memoryStorage();
const upload = multer({ storage: storage });

postRoutes.route('/management/photos/:orden', function(params, req, res) {
  const { orden } = params;
  
  authenticate(req, res, () => {
    upload.fields([
      { name: 'foto1', maxCount: 1 },
      { name: 'foto2', maxCount: 1 }
    ])(req, res, async function(err) {
      if (err) {
        res.writeHead(400, { 'Content-Type': 'text/plain' });
        res.end('Error uploading files');
        return;
      }

      // Get uploaded files
      const foto1 = req.files['foto1'] ? req.files['foto1'][0] : null;
      const foto2 = req.files['foto2'] ? req.files['foto2'][0] : null;

      // Save photos to collection
      const fotos = [foto1, foto2].map(photo => {
        if (!photo) return null;
        
        const fileId = Random.secret(10);

        photosCollection.write(photo.buffer, {
          fileName: photo.originalname,
          type: photo.mimetype,
          fileId
        });

        return fileId;
      });

      // Update record with photo IDs
      recordsCollection.update(
        { NUMERO_DE_LA_ORDEN: orden },
        { $set: { fotos } },
        { upsert: true }
      );

      res.writeHead(200, { 'Content-Type': 'text/plain' });
      res.end('Record added successfully');
    });
  });
});
```

## Retrieving Photos

Photos are retrieved using Meteor methods:

### Get File by ID

```javascript imports/api/photos/photosMethods.js
import { Meteor } from 'meteor/meteor';
import { photosCollection } from './photosCollection';

function replaceBaseUrl(link) {
  return link.replace('http://localhost/', process.env.ROOT_URL);
}

Meteor.methods({
  async getFileById(id) {
    return await photosCollection.find({ _id: id });
  },
  
  async getFileLink(id) {
    return photosCollection
      .find({})
      .fetch()
      .map(function(fileRef) {
        return {
          link: replaceBaseUrl(photosCollection.link(fileRef)),
          id: fileRef._id
        };
      })
      .filter(dataLink => dataLink.id == id);
  }
});
```

### Getting Photo Links in Reports

When generating reports, photo links are resolved:

```javascript imports/ui/components/reportView/index.jsx
const updatedResult = await result[0].data.map(async (record) => {
  const { ubicacion, period, fotos, ...update } = record;
  
  // Get photo links
  const photoPromises = fotos
    ? fotos.map(photoId => Meteor.callAsync('getFileLink', photoId))
    : null;
  
  const FOTOS = photoPromises
    ? await Promise.all(photoPromises)
        .then(photos => photos.flat(1).map(p => p.link))
        .catch(() => 'Error')
    : [];

  return {
    ...update,
    FOTOS,
    // ... other fields
  };
});
```

## Photo Storage

### Storage Location

Photos are stored on the filesystem:

```
/data/photos/
├── abc123xyz.jpg
├── def456uvw.png
├── ghi789rst.jpeg
└── ...
```

<Steps>
  <Step title="Create Storage Directory">
    ```bash
    sudo mkdir -p /data/photos
    sudo chown meteorus:meteorus /data/photos
    sudo chmod 755 /data/photos
    ```
  </Step>
  
  <Step title="Configure Collection">
    Storage path is set in photosCollection:
    
    ```javascript
    storagePath: '/data/photos'
    ```
  </Step>
  
  <Step title="Set Permissions">
    Ensure the application user has write access:
    
    ```bash
    ls -la /data/photos
    # Should show: drwxr-xr-x meteorus meteorus
    ```
  </Step>
</Steps>

## Photo Security

### Access Control

Photos require authentication:

```javascript
function authenticate(req, res, next) {
  const authHeader = req.headers['authorization'];
  const token = authHeader && authHeader.split(' ')[1];

  if (!token) {
    res.writeHead(401);
    res.end('Unauthorized: No token provided');
    return;
  }

  jwt.verify(token, JWT_SECRET, (err, decoded) => {
    if (err) {
      res.writeHead(403);
      res.end(JSON.stringify({ message: 'Invalid token' }));
      return;
    }

    const user = Meteor.users.findOne(decoded.userId);
    if (!user || user.profile.role !== 'management') {
      res.writeHead(403);
      res.end(JSON.stringify({ message: 'Access denied' }));
      return;
    }

    next();
  });
}
```

<Info>
Only authenticated users with the 'management' role can upload photos via the API.
</Info>

### File Validation

Photos are validated before upload:

1. **Size check**: Maximum 10MB
2. **Format check**: Only PNG, JPG, JPEG
3. **Extension verification**: Regex test on file extension

```javascript
function checkFile(size, regex_formats, file) {

  }
  if (!checkFormat(regex_formats, file)) {
    return 'Formato no válido, solo se aceptan imagenes';
  }
  return true;
}
```

## Troubleshooting

<Accordion title="Photo upload fails with 400 error">
  Check:
  1. File size is under 10MB
  2. File format is PNG, JPG, or JPEG
  3. Authorization token is valid
  4. Order number exists in database
  5. `/data/photos` directory has write permissions
</Accordion>

<Accordion title="Photos not visible in reports">
  Verify:
  1. Photo IDs are stored in record's `fotos` array
  2. Photos exist in `/data/photos` directory
  3. `ROOT_URL` environment variable is set correctly
  4. Photo links are being resolved properly
  5. Browser can access photo URLs
</Accordion>

<Accordion title="Permission denied when saving photos">
  Fix permissions:
  
  ```bash
  sudo chown -R meteorus:meteorus /data/photos
  sudo chmod -R 755 /data/photos
  ```
</Accordion>

<Accordion title="Photos taking too long to load">
  Consider:
  1. Implementing photo thumbnails
  2. Using lazy loading in UI
  3. Compressing images on upload
  4. Serving via CDN or reverse proxy cache
</Accordion>

## Migration to Cloud Storage

For production deployments, consider migrating to cloud storage:

### AWS S3 Example

```javascript
import AWS from 'aws-sdk';

const s3 = new AWS.S3({
  accessKeyId: process.env.AWS_ACCESS_KEY,
  secretAccessKey: process.env.AWS_SECRET_KEY
});

export const photosCollection = new FilesCollection({
  collectionName: 'photos',
  
  // Custom upload handler
  onAfterUpload(fileRef) {
    const params = {
      Bucket: 'movilidad-photos',
      Key: fileRef._id,
      Body: fs.createReadStream(fileRef.path),
      ContentType: fileRef.type
    };
    
    s3.upload(params, (err, data) => {
      if (err) {
        console.error('S3 upload error:', err);
      } else {
        // Update file reference with S3 URL
        photosCollection.update(fileRef._id, {
          $set: { 'meta.s3Url': data.Location }
        });
        
        // Optionally delete local file
        fs.unlink(fileRef.path, () => {});
      }
    });
  }
});
```
